What's in a Graph?

Graph Theory: A graph is an ordered pair G = (V, E) comprising a set V of vertices, nodes or points together with a set E of edges, arcs or lines, which are 2-element subsets of V

V = [A, B, C, D, E, F]

E = [(A, B), (A, C), (E, C)]

What is a Graph Database?

  • Nodes and relationships are first class
  • Local relationship based queries are first class

Why Use a Graph Database?

  • Data model is graph'ish
  • MOST queries for your data are graph'ish
  • Performance of graph'ish queries are IMPORTANT
  • Queries do not require a "complete node scan"

Performance Part 1

Given 1,000 users with an average of 50 "friend" relationships each, determine if one person is connected to another in 4 or fewer hops.

Why So Fast?

Nodes store references to their neighbors

Relational DB Log2n to look up neighbor

Cypher : Basic Nodes

CREATE a node

CREATE (n)

Find all nodes

MATCH (n) RETURN n;

Find node by id

START n = node(13) RETURN n;

Cypher : Basic Nodes

DELETE all nodes

MATCH (n) DELETE n;

DELETE single node

START n = node(13) DELETE n;

Cypher : Nodes With Labels

CREATE a node with a label

CREATE (n:Person);

CREATE a node with multiple labels

CREATE (n:Person:Manager);

Find all Person nodes

MATCH (n:Person) RETURN n;

DELETE all PERSON nodes

MATCH (n:Person) DELETE n;

Cypher : Nodes With Labels

CREATE a node with a label

CREATE (n:Person);

Find all Person nodes

MATCH (n:Person) return n;

Cypher : Nodes With Properties

CREATE a node with properties

CREATE (n:Person {name:"Bert"});

Find all Person nodes

MATCH (n:Person) return n;
